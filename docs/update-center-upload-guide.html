申悅更新中心上傳教學

這份文件說明兩種更新中心新增方式：App 內表單上傳到 Google Apps Script，以及在電腦上用 GitHub 工具發布正式 APK。

方式一：App 內新增應用

  1. 進入「更新」分頁，按「新增應用」。
  2. 輸入管理 PIN：7708。
  3. 選擇 APK 檔案；系統會自動讀取應用名稱、圖標、分類、介紹、容量、套件名稱、版本、SDK 與 SHA-256。
  4. 第一張圖片與第二張圖片可直接按「圖片上傳」選檔。
  5. 大型 APK 建議不要直接選檔，請先上傳到 GitHub Releases，再把下載網址貼到「應用下載地址」。
  6. 套件名稱與版本碼會由 APK 自動帶入；只有解析失敗時才需要手動補填。
  7. 按「儲存並上傳」。資料會送到 Google Apps Script，檔案會保存到 Google Drive。

注意：App 內表單適合圖片與小型 APK。正式大量 APK 建議使用方式二。

方式二：電腦上傳到 GitHub Releases

這是正式推薦流程。APK 會上傳到 SYLONG7708/update 的 apk-cloud Release，並更新 updates.json。

powershell -ExecutionPolicy Bypass -File .\tools\publish-update-app.ps1 `
  -ApkPath "C:\路徑\app.apk" `
  -Name "App 顯示名稱" `
  -PackageName "com.example.app" `
  -VersionCode 100 `
  -VersionName "1.0.0" `
  -Category "影音播放" `
  -Description "這裡填 App 介紹" `
  -IconPath "C:\路徑\icon.png" `
  -FirstImagePath "C:\路徑\photo-1.png" `
  -SecondImagePath "C:\路徑\photo-2.png"

如果電腦有 Android SDK 的 aapt，工具會嘗試自動讀取套件名稱、版本碼與版本名稱；讀不到時再手動填參數。

完成後，APK 下載網址會是 GitHub Release asset，適合車機下載安裝。

常見問題

點圖片上傳或 APK 上傳沒反應

舊版 APK 的 WebView 沒有檔案選擇器。新版已修正，需重新安裝新版申悅車機助手 APK。

新增後安裝按鈕不能按

請確認已選 APK，且畫面已自動帶入 套件名稱 與 版本碼。如果只選小 APK 檔案，需等 Apps Script 產生雲端下載網址後重新整理清單。

圖片破圖

請確認圖片網址可公開讀取，或使用本機 GitHub 工具把圖片放到本專案的 assets/update-* 目錄。
