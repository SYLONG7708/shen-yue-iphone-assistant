申悅更新中心上傳教學

這份文件說明更新中心新增與替換方式：公開頁一鍵新增/替換、App 內表單同步到低權限 Google Apps Script，以及在電腦上用工具自動發布正式 APK 並同步 Apps Script。

方式零：公開頁一鍵新增或替換 APK

  1. 先在電腦啟動 SYLONG7708/update 裡的 tools/start-update-uploader.ps1。
  2. 工具會顯示一個公開上傳網址；把網址給需要上傳的人即可。
  3. 打開網址後選「替換既有 APK」或「新增新的 APK」。
  4. 替換模式可指定清單中的 App；若沒有指定，系統會依 APK 的套件名稱自動比對現有更新項目。
  5. 新增模式可填 App 顯示名稱、分類、介紹、圖標網址與詳情圖片網址；也可以直接選圖標圖片，APK 新增完成後會自動套用。
  6. 替換模式可選清單中的 App 後直接選圖標圖片，系統會立即更換該 App 的圖標並同步更新中心。
  7. 選 APK 後，系統會自動上傳 GitHub Release 檔案、更新 updates.json，並同步 Apps Script 更新清單。

不要把 GitHub token 寫在網頁裡；公開頁只連到後端服務，GitHub 權限只留在啟動工具的電腦端。

方式一：App 內新增應用

  1. 進入「更新」分頁，按「新增應用」。
  2. 輸入管理 PIN：7708。
  3. 選擇 APK 檔案；系統會自動讀取應用名稱、圖標、分類、介紹、容量、套件名稱、版本、SDK 與 SHA-256。
  4. 第一張圖片與第二張圖片可直接按「圖片上傳」選檔。
  5. APK 請先上傳到 GitHub Releases，再把下載網址貼到「應用下載地址」。右側 APK 檔案只用來自動讀取版本與容量。
  6. 套件名稱與版本碼會由 APK 自動帶入；只有解析失敗時才需要手動補填。
  7. 按「儲存並上傳」。資料會送到 Google Apps Script JSON 儲存。

注意：低權限 Apps Script 不保存 APK 或圖片檔；檔案請使用公開網址，正式 APK 建議使用方式二。

方式二：電腦自動上傳正式 APK

這是正式推薦流程。APK 會上傳到 SYLONG7708/update 的 apk-cloud Release，更新 updates.json，並可同步寫入 Apps Script 工作表。

powershell -ExecutionPolicy Bypass -File .\tools\publish-update-app.ps1 `
  -ApkPath "C:\路徑\app.apk" `
  -Name "App 顯示名稱" `
  -PackageName "com.example.app" `
  -VersionCode 100 `
  -VersionName "1.0.0" `
  -Category "影音播放" `
  -Description "這裡填 App 介紹" `
  -IconPath "C:\路徑\icon.png" `
  -FirstImagePath "C:\路徑\photo-1.png" `
  -SecondImagePath "C:\路徑\photo-2.png" `
  -AppsScriptId "1HUOf9VUijyDLDCRrpGNJySVp-xuFvq7MWqUBVju3jPjxS7VnDgqmJdE7" `
  -DeployAppsScript `
  -SyncAppsScript

如果電腦有 Android SDK 的 aapt，工具會嘗試自動讀取套件名稱、版本碼與版本名稱；讀不到時再手動填參數。

完成後，APK 下載網址會是 GitHub Release asset，Apps Script 也會有同一筆更新資料，適合車機下載安裝。

第一次設定 Apps Script 自動部署

目前已完成低權限 Apps Script 部署。若修改 Code.gs，執行以下命令即可重新部署，不需要再要求 Google 敏感授權。

powershell -ExecutionPolicy Bypass -File .\tools\deploy-apps-script.ps1 `
  -ScriptId "1HUOf9VUijyDLDCRrpGNJySVp-xuFvq7MWqUBVju3jPjxS7VnDgqmJdE7" `
  -DeploymentId "AKfycbwrUCUeksZrWOUSDrdKgUGTS1JIPRX3c18PIKgZu_j64jBZGXjI7rnHTFjmIqUljZFzeg"

如果 Code.gs 已經部署過，正式發布時可省略 -DeployAppsScript，只使用 -SyncAppsScript 寫入更新項目。

常見問題

點圖片上傳或 APK 上傳沒反應

舊版 APK 的 WebView 沒有檔案選擇器。新版已修正，需重新安裝新版申悅車機助手 APK。

新增後安裝按鈕不能按

請確認已填 APK 下載地址，且畫面已自動帶入 套件名稱 與 版本碼。低權限 Apps Script 不會產生 Google Drive 下載網址。

圖片破圖

請確認圖片網址可公開讀取，或使用本機 GitHub 工具把圖片放到本專案的 assets/update-* 目錄。

Apps Script 顯示仍是舊版

執行 tools/deploy-apps-script.ps1 重新部署既有部署 ID。成功後工具會自動讀取 ?type=updates，確認回傳內容有 apps。
