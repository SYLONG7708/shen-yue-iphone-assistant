Warranty Apps Script Guide

申悅保固 Apps Script 部署與填寫教學

這份教學給完全不懂程式的人照著做。目標是讓「保固資料登錄」按下上傳後，資料真的寫進指定 Google 試算表，而不是只顯示成功。

重點：更新中心和保固資料使用不同 Apps Script。不要把 Library 網址貼到管理設定，也不要把保固網址拿去當更新中心網址。

第一步：確認保固 Apps Script 內容

  1. 打開保固 Apps Script 專案連結。
  2. 左邊選 編輯器，打開 Code.gs。
  3. 確認程式裡有 doGet、doPost，而且 doPost 會處理 iphone-warranty。
  4. 確認程式是用 SpreadsheetApp.getActiveSpreadsheet()，不要改成 openById()。目前這個保固專案是綁在試算表上的版本，用 currentonly 權限比較不容易被 Google 擋住。

如果你只是重新部署，不需要重貼程式；確認內容存在後直接做下一步。

第二步：確認 appsscript.json 權限

  1. 在 Apps Script 左側點 專案設定。
  2. 打開 在編輯器中顯示 appsscript.json 資訊清單檔案。
  3. 回到 編輯器，打開 appsscript.json。
  4. 確認內容包含下列設定。
{
  "timeZone": "Asia/Taipei",
  "runtimeVersion": "V8",
  "exceptionLogging": "STACKDRIVER",
  "oauthScopes": [
    "https://www.googleapis.com/auth/spreadsheets.currentonly"
  ],
  "webapp": {
    "executeAs": "USER_DEPLOYING",
    "access": "ANYONE_ANONYMOUS"
  }
}

不要改成完整 https://www.googleapis.com/auth/spreadsheets，那比較容易出現「系統已封鎖這個應用程式」。

第三步：重新部署 Web App

  1. 在 Apps Script 右上角按 部署。
  2. 如果是第一次或想保留舊版，選 新增部署作業。
  3. 齒輪或類型選單選 網頁應用程式。
  4. 說明填：warranty-sheet-write。
  5. 執行身分 選：我。
  6. 誰可以存取 選：任何人。
  7. 按 部署。
  8. 如果跳授權，選你的 Google 帳號並允許它存取目前這份試算表。
  9. 部署完成後，只複製 網頁應用程式網址，必須長得像 https://script.google.com/macros/s/.../exec。

不要複製 /macros/library/d/.../3，那是 Library 版本網址，不是 App 可以上傳資料的 Web App 網址。

第四步：管理設定三個欄位怎麼填

  1. 填完後按 儲存管理設定。
  2. 按 測試保固寫入，畫面應該顯示「保固測試已寫入 Google 試算表第 X 列」。
  3. 打開 Google 試算表，確認最後一列有「雲端測試」。
  4. 測試成功後，到保固頁面填正式客戶資料，再按 儲存並直接上傳。

如何判斷有沒有成功

成功

畫面顯示「保固資料已寫入 Google 試算表第 X 列」。

Google 試算表最後一列會出現車主姓名、電話、車牌、安裝項目、保固到期日。

失敗

畫面顯示「沒有回傳 Google Sheet 列號」或 HTTP 403。

代表網址填錯、部署權限錯、Google 授權未完成，或貼到的是更新中心 Apps Script 而不是保固 Apps Script。

常見錯誤

顯示「系統已封鎖這個應用程式」

這是 Google 阻擋 OAuth 授權，不是 App 按鈕壞掉。解法是使用目前這個已綁定試算表的保固 Apps Script，並維持 spreadsheets.currentonly。如果 Google 仍完全封鎖授權，無法用指令或網頁繞過，只能改 Google Cloud OAuth 驗證、換已授權帳號、或改用其他後端。

貼了 /macros/library/d/... 還是不能上傳

Library 網址不能拿來收 App 上傳資料。管理設定只接受 Web App URL，也就是結尾是 /exec 的網址。

更新中心突然讀不到清單

通常是把保固網址貼到更新中心欄位。更新中心欄位要用 AKfycbwr...，保固欄位才用 AKfycbwl...。

保固上傳顯示沒有 Google Sheet 列號

代表 Apps Script 沒有真的寫入 Sheet。請重新檢查 Web App URL、部署權限、以及 doPost 是否回傳 row。
